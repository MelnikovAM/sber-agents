# Conventions for Code Generation

- Всегда стремись к максимальной простоте и читаемости кода (KISS).
- Не реализуй то, что не требуется для текущей задачи (YAGNI).
- Пиши минимум кода для рабочего решения.
- Используй только необходимые зависимости и модули.
- Структурируй код по функциям; классы — только если это оправдано.
- Избегай абстракций, усложняющих понимание логики.
- Один файл — одна логическая зона ответственности, не дроби без необходимости.
- Все параметры и секреты — только через переменные окружения.
- Не реализуй сохранение данных, если оно не указано явно.
- Не добавляй обработку дополнительных кейсов и команд без запроса на это.
